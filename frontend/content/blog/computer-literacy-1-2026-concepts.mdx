## 결론 먼저

2026 컴활 1급은 **필기 60문항 / 60분 + 실기 Excel·Access 각 45분**, 출제기준은 2024–2026년 대한상공회의소 기준이 그대로 적용됩니다. 필기는 "컴퓨터 일반 / 스프레드시트 / 데이터베이스" 3과목, 실기는 Excel(고급함수·매크로·VBA) + Access(쿼리·폼·보고서) 복합형이에요. 이 글은 필기 과목별 핵심 개념과 실기에서 계속 나오는 함수·SQL·VBA 패턴을 2026 버전으로 묶어 정리한 요약본입니다.

## 왜 2026년에 다시 정리해야 하는가

2025년 말을 기점으로 정기 시험이 종료되고 2026년부터는 **상시 시험 중심**으로 운영돼요. 대한상공회의소 자격평가사업단 일정표를 보면 1급은 사실상 연중 내내 시험장 단위로 접수 창이 열리고 닫힙니다. 즉 "언제 응시할까"보다 "언제까지 끝낼까"가 중요해졌다는 뜻이에요.

출제기준은 2024–2026년용 공문서가 그대로 살아 있지만, 최근 1년(2024년 상시~2025년 상시) 기출을 보면 실기 Access의 **불일치 쿼리·Not In 서브쿼리**, Excel의 **배열수식·FREQUENCY**, VBA의 **이벤트 프로시저** 비중이 눈에 띄게 올라와 있어요. 그래서 기존 2020년대 초반 요약본으로 공부하면 실기에서 점수 구멍이 생깁니다.

<Highlight color="amber">2026 컴활 1급은 "필기 개념 암기 + 실기 1 회독 + 기출 3회 회독"이 최소 조합이에요. 실기를 빼고 필기만 반복하면 1과목은 되고 2·3과목에서 떨어지는 전형적인 패턴에 걸립니다.</Highlight>

## 1과목: 컴퓨터 일반 — 2026 출제 경향

기존의 장치·OS·네트워크 기본기 위에 **정보보안·멀티미디어 코덱·클라우드**가 최근 2년간 조금씩 늘어왔어요. 2026에도 이 흐름은 유지됩니다.

### 기억장치 계층과 캐시 적중률

```
레지스터 < 캐시(L1/L2/L3) < 주기억(RAM) < 보조기억(SSD/HDD) < 백업(테이프)
```

- 캐시 적중률 = 적중 횟수 / 전체 참조 횟수
- 평균 접근 시간 = H·Tc + (1−H)·Tm  (H=적중률, Tc=캐시시간, Tm=주기억시간)

<Highlight color="red">2026년에도 "평균 접근 시간 계산" 문제가 나올 확률이 높아요. 공식을 외우지 말고 "적중이면 캐시만, 실패면 캐시+주기억 둘 다 접근"으로 이해하세요.</Highlight>

### 정보보안 용어

| 용어 | 정의 |
|------|------|
| 스니핑 | 네트워크 패킷 도청 (수동적) |
| 스푸핑 | IP·MAC·DNS 위장 |
| 피싱/파밍 | 가짜 사이트 유도 / DNS 변조 |
| 랜섬웨어 | 파일 암호화 후 금전 요구 |
| 제로데이 | 패치 전 취약점 공격 |

### 멀티미디어 파일

- 이미지: JPEG(손실), PNG(비손실·투명도), GIF(애니메이션)
- 오디오: MP3(MPEG-1 Layer 3), AAC, WAV(무압축)
- 동영상: MP4, MKV, H.264/H.265(HEVC)

## 2과목: 스프레드시트 일반 — 2026 핵심

필기 2과목은 "Excel 기능 설명 문제 + 수식 결과 예측 문제"가 반반이에요. 이 중 실기로 연결되는 함수는 필기에서도 높은 확률로 지문에 나옵니다.

### 참조 방식 총정리

| 구분 | 표기 | `=A1+B$1` 을 오른쪽·아래로 복사 |
|------|------|------------------------------|
| 상대 | A1 | 행·열 모두 바뀜 |
| 절대 | $A$1 | 고정 |
| 혼합 | A$1 | 열만 바뀜, 행 고정 |
| 혼합 | $A1 | 행만 바뀜, 열 고정 |

### 2026 실기에서 반복 출제되는 Excel 수식 3개

**1) 조건부 순위(RANK + IF 배열수식)**

```excel
=RANK(D2, IF($B$2:$B$20=B2, $D$2:$D$20)) + 1 - 1
// Ctrl+Shift+Enter 로 배열수식 확정
// 의미: 같은 부서(B열) 내에서만 매출(D열) 순위
```

**2) 구간 빈도 집계(FREQUENCY)**

```excel
// 결과 범위(예: F2:F5) 전체 선택 후 입력, Ctrl+Shift+Enter
=FREQUENCY($C$2:$C$100, $E$2:$E$4)
// C열 점수를 E열 구간(예: 60,70,80)으로 나눠 각 구간 인원수 반환
```

**3) 다중 조건 조회(INDEX + MATCH 배열)**

```excel
=INDEX($D$2:$D$100,
  MATCH(1, ($A$2:$A$100=G2)*($B$2:$B$100=H2), 0))
// Ctrl+Shift+Enter
// A·B 두 조건을 동시에 만족하는 행의 D값
```

<Highlight color="amber">VLOOKUP은 "왼쪽 열에서만 검색" 제약이 있어서 2026 실기는 INDEX+MATCH 조합이 기본값이에요. VLOOKUP이 나와도 거의 "오답 보기"로 함께 등장합니다.</Highlight>

### 피벗·고급필터·시나리오

- **고급필터**: 조건 범위는 "필드명 + 조건" 2줄 이상, AND는 같은 행·OR는 다른 행
- **피벗**: 값 필드 설정에서 합계/평균/개수 외에 "값 표시 형식 → 열 합계 비율"이 단골
- **시나리오**: 변경 셀과 결과 셀을 먼저 정하고, 요약 보고서는 새 워크시트로 생성

## 3과목: 데이터베이스 일반 — 2026 업그레이드 포인트

예전에는 "용어 정의" 위주였는데 2024년부터 **정규화 단계별 판단**, **ER 다이어그램 → 릴레이션 변환**, **관계 대수 vs SQL 대응**이 섞여 나오기 시작했어요. 2026도 동일합니다.

### 정규화 단계 판단

| 단계 | 제거 대상 | 기억법 |
|------|----------|--------|
| 1NF | 반복 그룹(복합 값) | "원자값만" |
| 2NF | 부분 함수 종속 | "기본키 전체에 종속" |
| 3NF | 이행 함수 종속 | "A→B→C 끊기" |
| BCNF | 결정자가 후보키가 아닌 종속 | "모든 결정자 = 후보키" |

### 키 vs 제약조건

| 항목 | 핵심 |
|------|------|
| 기본키 | 유일성 + 최소성 + NOT NULL |
| 후보키 | 유일성 + 최소성 |
| 슈퍼키 | 유일성만 |
| 외래키 | 참조 무결성 — 참조되는 쪽 PK 삭제 시 CASCADE / SET NULL / RESTRICT |

### 관계 대수 ↔ SQL

```sql
-- σ 조건 (R)    =  SELECT * FROM R WHERE 조건
-- π 속성 (R)    =  SELECT 속성 FROM R
-- R ⋈ S         =  SELECT * FROM R NATURAL JOIN S
-- R ÷ S         =  존재량 비교 (NOT EXISTS ... NOT EXISTS ...)
```

## 실기: Access 쿼리 2026 필수 패턴

최근 상시 기출을 보면 **불일치 쿼리**, **매개 변수 쿼리**, **크로스탭**이 순환 출제돼요.

**1) 불일치 쿼리(한쪽에만 있는 값 찾기)**

```sql
-- 주문은 있는데 고객 마스터에는 없는 고객ID
SELECT O.고객ID, O.주문일자
FROM 주문 AS O
LEFT JOIN 고객 AS C ON O.고객ID = C.고객ID
WHERE C.고객ID IS NULL;
```

`NOT IN` 서브쿼리로도 풀 수 있지만 실기 채점은 결과 테이블이 같으면 맞춰주니 둘 중 편한 쪽을 선택하세요. 단 `NOT IN`에 NULL이 섞이면 결과가 비는 함정이 있습니다.

**2) 그룹 집계 + HAVING**

```sql
-- 2026년 상반기 기준, 부서별 평균 매출이 500 이상인 부서만
SELECT 부서, AVG(매출) AS 평균매출
FROM 실적
WHERE 연도 = 2026 AND 분기 IN (1, 2)
GROUP BY 부서
HAVING AVG(매출) >= 500
ORDER BY 평균매출 DESC;
```

<Highlight color="red">WHERE는 그룹화 "전", HAVING은 그룹화 "후" — 이 순서를 바꾸면 집계 결과가 완전히 달라져요. 2026 필기·실기 모두에서 이 차이를 묻습니다.</Highlight>

## 실기: VBA 매크로 — 2026 신기출 패턴

2024~2025년에 이벤트 프로시저(폼 열기, 버튼 클릭, Worksheet_Change)의 비중이 꾸준히 올라왔어요. "매크로 기록"만으로는 풀 수 없는 유형이 실기 Excel 4번 문항으로 종종 나옵니다.

```vb
' [예시] 특정 범위 입력 시 자동으로 서식 적용하는 워크시트 이벤트
Private Sub Worksheet_Change(ByVal Target As Range)
    ' B열에 숫자 입력되면 1000 이상은 빨강, 아니면 검정
    If Not Intersect(Target, Range("B2:B100")) Is Nothing Then
        Application.EnableEvents = False
        If IsNumeric(Target.Value) Then
            If Target.Value >= 1000 Then
                Target.Font.Color = RGB(255, 0, 0)
            Else
                Target.Font.Color = RGB(0, 0, 0)
            End If
        End If
        Application.EnableEvents = True
    End If
End Sub
```

시험장에서는 **`Application.EnableEvents = False`를 빼먹으면 무한 루프**에 걸려요. 내 코드가 셀을 바꾸는 순간 Worksheet_Change가 자기 자신을 다시 호출하기 때문입니다. 이 한 줄이 2026 실기 VBA 문항의 가장 큰 감점 포인트예요.

## 자주 하는 실수 지점 (2026 기준)

- **배열수식에 Ctrl+Shift+Enter 누락** — Excel 365/2021의 동적 배열 때문에 집에서는 엔터만 눌러도 되는데 시험장은 2021 LTSC라 구버전 동작이라 CSE 필요
- **Access 조건에 한글 따옴표** — `"서울"` 대신 `"서울"`(스마트 따옴표)이 들어가면 결과 0건
- **VLOOKUP 4번째 인수 생략** — 2026 필기 최근 기출도 여전히 단골, TRUE/FALSE 의미를 반대로 기억하지 말 것
- **정규화 2NF/3NF 혼동** — "부분 종속"과 "이행 종속"을 기본키 구성으로 판단 (복합키 → 2NF, 단일키 → 보통 3NF만 따짐)
- **VBA에서 `=` 하나만 씀** — 비교와 대입 모두 `=`지만, If 안의 비교는 `Target.Value = 1000`, 대입은 `Target.Value = 1000` 처럼 "왼쪽이 변수냐 조건이냐"로 구분

## 시험장에서의 접근법

1. **필기는 3과목 → 2과목 → 1과목 순서로 푼다** — 데이터베이스가 난이도가 제일 높아 에너지 많을 때 처리, 1과목(암기)은 마지막 5분에도 가능
2. **실기 Excel 45분은 "기본 작업 10분 + 계산 15분 + 분석 10분 + 매크로 10분"**로 쪼개기 — 매크로·VBA에 30분을 쓰면 다른 문항을 날립니다
3. **Access 45분은 "테이블 설정 5분 + 쿼리 15분 + 폼 15분 + 보고서 10분"** — 쿼리에서 막히면 바로 보고서로 넘어가 부분 점수 확보
4. **모르는 함수가 나오면 F1 헬프 사용 가능** — 시험장 PC에 Office 도움말이 살아 있으므로 함수 인수 순서는 그 자리에서 확인
5. **마지막 3분은 저장 확인** — 파일명 변경 금지, `Ctrl+S`만
