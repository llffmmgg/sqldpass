## SQLD가 뭔가요?

SQLD(SQL Developer)는 한국데이터산업진흥원에서 주관하는 국가공인 자격증입니다. 데이터베이스와 SQL에 대한 기본 역량을 검증하는 시험으로, IT 취업을 준비하는 분들에게 가장 인기 있는 자격증 중 하나입니다.

**시험 구성:**
- **1과목**: 데이터 모델링의 이해 (10문항)
- **2과목**: SQL 기본 및 활용 (40문항)
- **총 50문항**, 객관식 4지선다
- **합격 기준**: 총점 60점 이상 + 과목별 40% 이상

---

## 과목별 학습 전략

### 1과목: 데이터 모델링의 이해

10문항밖에 안 되지만 방심하면 안 됩니다. 과목 과락(40%)이 있기 때문에 4문제 이상은 반드시 맞춰야 합니다.

**핵심 주제:**
- 데이터 모델의 이해 (개념적/논리적/물리적)
- 엔터티, 속성, 관계
- 식별자 (주식별자, 외부식별자)
- 정규화 (1NF ~ 3NF, BCNF)
- 반정규화

**공부 팁:**
- 개념을 이해하는 것보다 **용어를 정확히 외우는 게 중요**합니다
- 정규화 문제는 거의 매번 출제됩니다 — 각 정규형의 조건을 확실히 구분하세요
- ERD 읽는 법을 반드시 익히세요

### 2과목: SQL 기본 및 활용

40문항으로 시험의 대부분을 차지합니다. 여기서 점수를 확보해야 합니다.

**핵심 주제:**
- SELECT, WHERE, GROUP BY, HAVING, ORDER BY
- JOIN (INNER, LEFT, RIGHT, FULL, CROSS)
- 서브쿼리 (스칼라, 인라인뷰, 중첩)
- 윈도우 함수 (ROW_NUMBER, RANK, LAG, LEAD)
- DDL, DML, DCL, TCL
- 계층형 질의 (START WITH, CONNECT BY)
- 옵티마이저와 인덱스 기초

**공부 팁:**
- SQL 문법은 **직접 쳐보는 게 가장 빠릅니다**
- JOIN과 서브쿼리는 매 시험 5문제 이상 출제됩니다
- 윈도우 함수는 최근 출제 비중이 높아지고 있습니다
- NULL 처리 관련 문제는 함정이 많으니 주의하세요

---

## 효과적인 학습 순서

1. **이론 1회독** (3~5일) — 노범석 교재 또는 강의로 전체 개념 파악
2. **기출 분석** (2~3일) — 최근 5회분 기출을 풀면서 출제 패턴 파악
3. **취약 영역 보강** (2~3일) — 틀린 문제의 주제를 집중 학습
4. **모의고사 반복** (시험 전까지) — CBT로 실전 연습

## CBT 모의고사 활용법

### 왜 CBT인가?

실제 SQLD 시험은 CBT(Computer Based Test)로 진행됩니다. 종이 시험과 화면에서 문제를 푸는 것은 체감이 다릅니다. 특히:

- 화면에서 SQL 코드를 읽는 속도가 다릅니다
- 스크롤하면서 문제를 파악하는 연습이 필요합니다
- 남은 시간을 화면에서 확인하는 습관을 들여야 합니다

### 모의고사 200% 활용하기

1. **시간을 재면서 풀기** — 50문항 60분. 한 문제당 1분 12초입니다.
2. **오답 노트 활용** — 틀린 문제를 다시 풀고, 2번 연속 맞추면 마스터 처리됩니다.
3. **카테고리별 풀기** — 약한 과목만 집중적으로 연습할 수 있습니다.
4. **여러 회차 풀기** — 매번 새로운 문제로 구성되기 때문에 답을 외울 걱정이 없습니다.

## 시험 당일 팁

- **1과목은 빠르게** — 20분 이내로 끝내고 2과목에 시간을 투자하세요
- **모르는 문제는 표시하고 넘기기** — CBT에서는 문제 마킹 기능이 있습니다
- **NULL 문제는 천천히** — NULL 관련 함정이 가장 많습니다
- **시간 관리** — 마지막 10분은 마킹한 문제를 다시 보는 시간으로 남기세요
