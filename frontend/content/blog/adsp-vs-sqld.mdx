## 결론부터: 둘은 완전 다른 시험이에요

ADsP와 SQLD 모두 **한국데이터산업진흥원(K-DATA)** 이 주관하는 국가공인 자격증이라 비슷해 보이지만, **실제로는 과목도 난이도도 검증 방향도 다릅니다**.

"둘 다 데이터 관련이니까 하나만 따도 되나?"라는 질문을 많이 받는데, 답은 **"목표 직무에 따라 다르다"** 예요.

---

## 한눈에 비교

| 항목 | SQLD | ADsP |
|------|------|------|
| 정식 명칭 | SQL 개발자 | 데이터분석 준전문가 |
| 주관 | 한국데이터산업진흥원 | 한국데이터산업진흥원 |
| 총 문항 | 50문항 | 50문항 |
| 시험 시간 | 90분 | 90분 |
| 과목 | 2과목 (모델링 / SQL) | 3과목 (이해 / 기획 / 분석) |
| 핵심 영역 | **SQL 실전 감각** | **데이터 분석 지식의 폭** |
| 문제 형식 | 4지선다 | 4지선다 (2024 개편 후) |
| 합격 기준 | 총점 60 + 과목별 40점 | 동일 |
| 응시료 | 5만원대 | 5만원대 |
| 합격률 (대략) | 50% 전후 | 60% 전후 (개편 후 하락 평가) |
| 상위 자격 | **SQLP** (전문가) | **ADP** (전문가) |

둘 다 비슷한 구조지만, **검증 대상이 다릅니다**. SQLD는 실제 SQL을 읽고 쓸 줄 아는지를 보고, ADsP는 데이터 분석 프로세스와 알고리즘 개념을 아는지를 봐요.

## SQLD는 뭘 검증하나

**"SQL을 실제로 다룰 수 있는가"** 를 봅니다.

- 1과목: 데이터 모델링의 이해 (ERD·정규화·키)
- 2과목: SQL 기본 및 활용 (JOIN·서브쿼리·윈도우 함수·NULL)

시험 문제가 거의 **실제 SQL 코드**로 나와요. JOIN 결과 예측, 서브쿼리 해석, NULL 함정 같은 걸 풀어야 합니다. 교재만 읽으면 안 되고, **SQL을 직접 쳐보면서 감각을 익혀야** 붙는 시험이에요.

**이런 사람에게 추천:**
- 백엔드·개발 취업 준비 중
- SI·금융·공공 개발자 희망
- DB 실무 역량이 필요한 직무

---

## ADsP는 뭘 검증하나

**"데이터 분석 전반을 이해하고 있는가"** 를 봅니다.

- 1과목: 데이터 이해 (빅데이터·DIKW·데이터 사이언티스트)
- 2과목: 데이터 분석 기획 (방법론·프로젝트 관리·거버넌스)
- 3과목: 데이터 분석 (**R·통계·회귀·분류·군집·연관**)

3과목이 30문항으로 전체의 60%예요. R 코드·통계 개념·머신러닝 알고리즘 개념까지 커버하는 **개념 시험**에 가깝습니다. SQL은 거의 안 나와요.

**이런 사람에게 추천:**
- 데이터 분석·사이언티스트 취업 준비 중
- 마케팅·기획 등 **비개발 직무에서 데이터 리터러시** 필요
- AI·통계 대학원 진학 준비 중 이력 한 줄 필요한 경우
- 상위 자격증인 **ADP** 준비 전 입문용

## 직무별 추천

| 목표 | 우선순위 1 | 우선순위 2 |
|------|----------|----------|
| 백엔드 개발자 | **SQLD** | 정처기 |
| SI·금융 개발 | **SQLD** | 정처기 |
| 데이터 분석가 | **ADsP** | SQLD |
| 데이터 사이언티스트 | **ADsP** → ADP | SQLD |
| 마케팅·기획 + 데이터 | **ADsP** | SQLD (부가) |
| 공기업·공공 | **SQLD** + 정처기 | (ADsP는 부가) |

<Highlight color="amber">데이터 분석 직무 준비한다면 ADsP 하나만으로는 이력서에 부족해요. SQLD·파이썬·태블로 같은 실무 스킬과 함께 묶어야 경쟁력이 생깁니다.</Highlight>

---

## 둘 다 딸 가치가 있나?

<Highlight color="green">있습니다. 특히 데이터 직무 지망생이라면 둘 다 권장합니다.</Highlight>

- SQLD는 **실전 SQL** 검증
- ADsP는 **분석 사고** 검증

두 자격증이 보는 영역이 겹치지 않아서 이력서에 같이 적으면 "DB도 알고 분석도 아는 사람"으로 보이는 효과가 있어요.

### 추천 순서

**데이터 분석 직무 지망:** ADsP → SQLD (분석 관점 먼저 잡고 SQL 실전으로)

**개발 직무 지망:** SQLD → (여유 있으면) ADsP

**둘 다 처음:** SQLD부터. ADsP는 범위가 넓어서 처음 데이터 자격증으로는 부담이에요.

---

## 준비 기간 차이

| 자격증 | 전공자·경험자 | 비전공자 |
|--------|--------------|---------|
| SQLD | 1–2주 | 3–4주 |
| ADsP | 2주 | 3–4주 (통계 배경 부족 시 1달+) |

SQLD는 SQL을 직접 쳐보면 감각이 빨리 오는데, ADsP는 개념 외울 게 많아서 **단기 속성은 어려워요**. 특히 통계 기초가 없으면 3과목이 부담됩니다.

## 함정 주의

### 1. "ADsP = 쉬운 버전 SQLD"라는 오해

**전혀 아니에요.** 과목 자체가 달라요. ADsP에서는 SQL 문제가 거의 안 나오고, SQLD에서는 통계·알고리즘이 안 나와요. 서로 대체재가 아니라 **보완재**입니다.

### 2. "ADsP가 더 쉬우니까 그것부터"

2024 개편 후 난이도가 올라가서 "ADsP가 더 쉽다"는 말이 항상 맞진 않아요. 오히려 비전공자 입장에선 **SQLD가 더 직관적**일 수 있습니다. SQL은 쳐보면 이해되지만 통계는 개념부터 쌓아야 하거든요.

### 3. "둘 다 따면 취업된다"

자격증 2개로 취업이 결정되진 않아요. **포트폴리오 + 스킬 + 자격증** 세트가 돼야 의미가 있습니다. 자격증은 "이 사람은 최소한 이 정도는 안다"의 증빙일 뿐이에요.

---

## 정리

SQLD와 ADsP는 이름만 비슷한 다른 시험이에요. <Highlight color="green">**목표 직무에 따라 순서를 정하고, 여유가 되면 둘 다 취득하는 것**</Highlight>이 이상적입니다. 데이터 분석 직무라면 ADsP 먼저, 개발 직무라면 SQLD 먼저 잡으세요.
