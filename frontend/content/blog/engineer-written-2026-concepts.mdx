## 결론 먼저

2026 정보처리기사 필기는 **출제 기준이 NCS 기반으로 한 번 더 다듬어졌고**, 기존 5과목 골격(소프트웨어 설계 / 소프트웨어 개발 / 데이터베이스 구축 / 프로그래밍 언어 활용 / 정보시스템 구축 관리)은 그대로지만, 최근 2~3년 동안 **신기술 용어·보안 모델·클라우드 아키텍처** 쪽으로 비중이 눈에 띄게 이동했어요. 이 글은 2026년 1회·2회 필기를 준비하는 사람이 **"2026에서 바뀐 포인트 + 5과목 핵심 개념 + 신기술 용어표"** 를 한 번에 정리하도록 쓴 요약본이에요.

---

## 왜 2026을 따로 정리해야 하는가

정보처리기사는 2020년 대개편 이후로 과목 구성은 굳어졌지만, **출제 용어와 사례는 매년 조용히 업데이트**되고 있어요. 2024·2025 기출을 보면 예전에는 보이지 않던 **제로 트러스트(Zero Trust), 마이크로서비스(MSA), 컨테이너/쿠버네티스, SDN, NFV, 블록체인, DPI, 양자암호, LLM·생성형 AI 영역의 용어**가 5과목 정보시스템 구축 관리 파트에 들어오고 있어요. 2026 시험을 준비하면서 2020년 초판 요약본만 돌리면, 20문항 중 2~3문항이 "본 적 없는 용어"로 나와 과락 라인을 위협받는 구조예요.

게다가 2026년은 **필기가 여전히 CBT 방식**이고, 연 3회 시행돼요(1회 1월말~3월초, 2회 5월, 3회 8월~9월초). 회차별로 문제은행이 다르기 때문에, **한 번에 5과목 개념을 얇게 여러 번 훑는 게** 회차 변동성 대응에 가장 유리해요.

## 1과목: 소프트웨어 설계 — 2026 포인트

### 한 줄 정의부터

소프트웨어 설계는 "요구사항을 **구조(아키텍처)와 모델(UML)로 바꾸는 과정**"이에요. 2026에서도 UML 분류·디자인 패턴·아키텍처 스타일은 고정 출제고, **MSA·이벤트 기반 아키텍처** 같은 최신 스타일 용어가 덧붙어요.

### 꼭 아는 개념 5가지

| 개념 | 한 줄 정리 | 2026 출제 포인트 |
|------|-----------|-----------------|
| **UML 다이어그램** | 행위(Use Case, Sequence, Activity, State) vs 구조(Class, Component, Deployment) | "이건 행위냐 구조냐" 분류 문제 |
| **GoF 디자인 패턴** | 생성(Singleton, Factory)·구조(Adapter, Proxy)·행위(Observer, Strategy) | 설명→패턴명 매칭 |
| **아키텍처 스타일** | 계층형, 클라이언트-서버, 파이프-필터, **MSA**, 이벤트 기반 | MSA vs 모놀리식 비교 |
| **요구사항 분석 기법** | 인터뷰, 설문, **유스케이스 다이어그램**, 워크숍, 브레인스토밍 | 기능/비기능 요구사항 구분 |
| **소프트웨어 품질 특성 (ISO 25010)** | 기능 적합성, 성능 효율성, 사용성, 신뢰성, 보안성, 호환성, 유지보수성, 이식성 | ISO 9126 → 25010 전환 |

<Highlight color="amber">MSA(Microservice Architecture): 서비스 단위로 쪼개 독립 배포. 단일 장애 영향 축소·기술 스택 자유도 향상이 장점, 분산 트랜잭션·운영 복잡도가 단점. 2025 기출에 "MSA 특징으로 옳지 않은 것"이 나왔어요.</Highlight>

### 자주 하는 실수

- UML "패키지" 다이어그램을 행위로 오답 체크 → 구조예요.
- "유즈케이스 기술서"와 "유즈케이스 다이어그램"을 같은 걸로 봄 → 기술서는 문서, 다이어그램은 그림이에요.

## 2과목: 소프트웨어 개발 — 2026 포인트

### 한 줄 정의부터

이 과목은 "**짠 뒤 합치고, 테스트하고, 배포하고, 형상을 관리한다**"의 묶음이에요. 자료구조 · 알고리즘 · 테스트 · 형상관리 · 인터페이스 구현이 축이고, 2026에서는 **CI/CD, Git 브랜치 전략, 컨테이너 패키징**이 용어 표에 슬쩍 섞여 나와요.

### 핵심 개념 표

| 주제 | 짧은 정의 | 헷갈리는 짝 |
|------|----------|-------------|
| **단위 / 통합 / 시스템 / 인수 테스트** | 모듈 → 결합 → 전체 → 사용자 수용 | 누가 하느냐: 개발자 → 테스터 → 사용자 |
| **상향식 vs 하향식 통합** | 하위부터 올라옴 / 상위부터 내려옴 | 드라이버(상향식) vs 스텁(하향식) |
| **화이트박스 vs 블랙박스** | 내부 로직 기반 / 입력-출력 기반 | 경로 커버리지 vs 동등 분할 |
| **자료구조** | 스택(LIFO), 큐(FIFO), 트리, 그래프, 해시 | 스택 후위표기법 계산은 거의 매 회차 |
| **알고리즘 복잡도** | 시간·공간 복잡도, Big-O | 버블 O(n²), 퀵 평균 O(n log n) |

### 2026에서 조심할 포인트

- **형상 관리 절차**: 식별 → 통제 → 감사 → 기록. 이 순서 그대로 외워 두세요.
- **패키징 도구·배포**: Docker, Helm, CI/CD 파이프라인 흐름이 개념 수준으로 출제돼요. "무중단 배포 방식(롤링, 블루-그린, 카나리)" 매칭이 2025 2회차에 나왔어요.
- **인터페이스 구현**: JSON vs XML, REST vs SOAP, **AJAX**의 비동기 특성.

## 3과목: 데이터베이스 구축 — 2026 포인트

### 한 줄 정의부터

"**관계형 DB를 어떻게 잘 쪼개고(정규화), 어떻게 안전하게(트랜잭션/ACID) 굴리느냐**"가 주제예요. 이 과목은 실기와도 맞닿아 있어서 2026 필기에서도 점수 확보 과목으로 봐요.

### 반드시 챙길 개념

| 개념 | 한 줄 정리 | 시험 포인트 |
|------|-----------|------------|
| **정규화 1NF~BCNF** | 원자값 → 부분함수종속 제거 → 이행함수종속 제거 → 결정자 = 후보키 | 조건만 맞추면 점수 |
| **키 종류** | 후보키, 기본키, 대체키, 슈퍼키, 외래키 | 슈퍼키 ⊇ 후보키 ⊇ 기본키 |
| **트랜잭션 ACID** | 원자성·일관성·독립성·영속성 | 상황→속성 매칭 |
| **SQL 실행 순서** | FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY | WHERE는 집계 전, HAVING은 집계 후 |
| **이상(Anomaly) 현상** | 삽입·삭제·갱신 이상 | 정규화가 풀어주는 문제 |

<Highlight color="sky">BCNF: 모든 결정자(Determinant)가 후보키면 BCNF. 3NF에서 "결정자가 후보키가 아닌" 함수 종속이 하나라도 남아 있으면 BCNF가 아니에요.</Highlight>

### 2026 신기술 끼어드는 지점

- **NoSQL 분류**: Key-Value(Redis), Document(MongoDB), Column(Cassandra), Graph(Neo4j). "문서형 DB의 대표 제품" 짝 맞추기.
- **CAP 이론**: 일관성(Consistency)·가용성(Availability)·분할 허용(Partition Tolerance) 중 2개만 동시 달성.
- **데이터 웨어하우스 vs 데이터 레이크**: 정형 vs 비정형까지 수용.

## 4과목: 프로그래밍 언어 활용 — 2026 포인트

### 한 줄 정의부터

"**C·Java·Python 코드 실행 결과 + 운영체제·네트워크 기초**"가 섞인 과목이에요. 코드 실행형 문항이 매 회차 6~8개 나와서, 손으로 따라가는 연습이 필수예요.

### 헷갈리기 쉬운 표

| 언어 | 포인터/참조 | 형변환 | 반복문 특이점 |
|------|-------------|-------|---------------|
| **C** | 포인터 명시 (`*p`, `&a`) | 명시적 캐스팅 | `for(i=0; i<n; i++)` |
| **Java** | 참조형(객체), 원시형 구분 | `(int)`, `Integer.parseInt` | 향상된 for문 |
| **Python** | 모든 것이 객체, 참조 기반 | 동적 타입 | `for x in range(n)` |

### OS·네트워크 핵심

- **프로세스 상태**: 생성 → 준비 → 실행 → 대기 → 종료
- **스케줄링**: FCFS(비선점), SJF(비선점), SRT(선점), Round Robin(선점, 타임 퀀텀), Priority
- **페이지 교체**: FIFO, LRU, LFU, **Optimal(이상적)**. 특히 LRU 계산 문제는 단골.
- **OSI 7계층**: 물리 · 데이터링크 · 네트워크 · 전송 · 세션 · 표현 · 응용. 각 계층의 대표 장비(리피터/브리지/라우터/게이트웨이)까지.
- **TCP vs UDP**: 연결형·신뢰성 vs 비연결형·속도.

### 자주 하는 실수

- Java에서 `String` 비교를 `==` 로 한 코드의 결과 예측 오류 → 참조 비교임을 기억.
- C에서 `sizeof` 값 계산할 때 포인터 크기(보통 4 또는 8바이트)를 잊는 경우.

## 5과목: 정보시스템 구축 관리 — 2026 핵심 변동 구간

### 한 줄 정의부터

"**요구 분석부터 운영·보안까지의 관리 용어 + 신기술 용어집**"이에요. 2026 개정 흐름에서 **가장 많이 바뀐 과목**이고, 공부 안 한 용어가 튀어나오면 그대로 실점이에요.

### 개발 방법론·비용 산정

| 개념 | 한 줄 정리 | 포인트 |
|------|-----------|--------|
| **SDLC 모델** | 폭포수, 프로토타입, **나선형**, V-모델, **애자일** | 위험 분석 = 나선형 |
| **애자일** | XP, Scrum, Kanban, Lean | 스크럼 용어: 스프린트, 백로그, 데일리, 회고 |
| **비용 산정** | LOC, 기능점수(FP), **COCOMO**(Organic/Semi-detached/Embedded) | COCOMO 모드 구분 |
| **PM 도구** | PERT, CPM, 간트 차트 | PERT는 확률, CPM은 결정 |

### 2026 신기술·보안 용어표 (가장 중요)

| 분류 | 용어 | 짧은 정의 |
|------|------|----------|
| **네트워크·인프라** | **SDN** | Software Defined Network, 제어 평면과 데이터 평면 분리 |
| | **NFV** | Network Functions Virtualization, 네트워크 기능 소프트웨어화 |
| | **DPI** | Deep Packet Inspection, 페이로드까지 검사 |
| | **CDN** | Content Delivery Network, 캐시 서버 분산 |
| **클라우드** | **IaaS/PaaS/SaaS** | 인프라·플랫폼·소프트웨어 서비스 계층 |
| | **MSA / 컨테이너** | 쪼갠 서비스를 Docker + Kubernetes로 운영 |
| | **서버리스(FaaS)** | 함수 단위 실행, 자원 관리 위임 |
| **보안 모델** | **제로 트러스트** | "절대 신뢰하지 않고 항상 검증" |
| | **DLP** | Data Loss Prevention, 데이터 유출 방지 |
| | **SIEM** | 보안 이벤트 수집·상관분석 |
| | **CC 인증 / ISMS-P** | 제품·조직 단위 보안 인증 |
| **암호·인증** | **AES, RSA, ECC, SHA-256** | 대칭·비대칭·해시 대표 알고리즘 |
| | **양자암호(QKD)** | 양자 키 분배 기반 |
| | **블록체인** | 분산원장, PoW/PoS 합의 |
| **데이터·AI** | **빅데이터 3V/5V** | Volume·Velocity·Variety(+Veracity·Value) |
| | **LLM / 생성형 AI** | 대규모 언어 모델, 프롬프트·RAG |
| | **메타버스·디지털 트윈** | 가상공간 모델링, 현실 반영 |

<Highlight color="violet">제로 트러스트(Zero Trust): 내부망도 신뢰하지 않고, 모든 접근을 사용자·기기·컨텍스트 단위로 검증. 2025년 2회 필기에 "제로 트러스트의 원칙으로 옳지 않은 것"이 직접 출제됐어요.</Highlight>

### 보안 공격 · 대응 짝

| 공격 | 한 줄 | 대응 |
|------|------|------|
| **SQL Injection** | 입력으로 SQL 조작 | 파라미터 바인딩, 입력 검증 |
| **XSS** | 스크립트 삽입 | 출력 이스케이프, CSP |
| **CSRF** | 인증 세션 탈취 요청 | CSRF 토큰, SameSite 쿠키 |
| **DDoS** | 대량 트래픽 | WAF, 스크러빙, Anti-DDoS |
| **랜섬웨어** | 파일 암호화 후 요구 | 백업, EDR |
| **APT** | 장기 지능형 지속 공격 | SIEM, 제로 트러스트 |

## 자주 하는 실수 모음

- **암기만 하고 기출을 안 품**: 2026 개념 정리를 아무리 해도 기출 3회독 없이는 점수로 안 이어져요. 개념 정리 → 회차별 기출 → 오답 정리가 한 세트예요.
- **신기술 용어를 한 번만 훑고 넘김**: SDN·NFV·제로 트러스트처럼 처음 보는 용어는 눈으로만 읽으면 시험장에서 보기 4개가 다 비슷해 보여요. 한 줄 정의를 직접 써 보세요.
- **4과목 코드 문제를 눈으로만 읽음**: Java `String.equals`, C 포인터, Python 슬라이싱은 연필로 한 줄씩 값을 추적해야 실수가 안 나요.
- **과락 방어를 놓침**: 과목별 40점 아래가 하나라도 있으면 평균 60점이 넘어도 불합격이에요. 1·5과목처럼 "암기 과목"을 버리지 마세요.

## 시험장에서의 접근법

- **1·5과목부터 치우기**: 용어 매칭 문제가 많아 10초 안에 답이 나오는 문항이 많아요. 여기서 시간을 확보해 두세요.
- **3과목(DB)은 가운데**: SQL·정규화는 한 번 흐름을 잡으면 빠르게 풀려요.
- **4과목(프로그래밍)은 마지막**: 코드 추적에 시간이 가장 많이 들어가요. 체력 세이브 후 집중.
- **애매한 문제는 30초 마킹 → 넘기기**: 과락 방어가 우선이에요. 돌아올 시간을 마지막 10분에 확보해 두세요.
- **2026 신기술 용어가 나오면 당황 금지**: 보기 4개 중 "가장 맞아 보이는 것"과 "가장 아닌 것"의 간극을 먼저 보는 게 낫아요.

- [ ] SQL 실행 순서
