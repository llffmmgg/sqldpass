## 정처기 실기, 왜 어렵다고 느끼는 걸까?

정보처리기사 실기는 필기와 완전히 다릅니다. 필기가 4지선다 객관식이라면, 실기는 **단답형과 서술형**으로 직접 답을 적어야 합니다.

"대충 찍어서 맞는" 전략이 통하지 않기 때문에 정확한 개념 암기가 필수입니다.

**시험 구성:**
- **총 20문항** (100점 만점)
- 단답형: 한 단어~한 줄 수준의 답
- 서술형(약술형): 2~3문장 수준의 답
- **합격 기준**: 60점 이상

---

## 출제 경향 분석

최근 시험을 분석하면 몇 가지 뚜렷한 패턴이 보입니다.

### 프로그래밍 (배점 높음)

거의 매 시험 **4~6문제**가 코드 실행 결과를 묻습니다.

- **C언어**: 포인터, 배열, 구조체, 재귀함수
- **Java**: 상속, 오버라이딩, 인터페이스, 추상클래스
- **Python**: 리스트 조작, 슬라이싱, 딕셔너리

> 코드 트레이싱(한 줄씩 따라가며 결과 추적)은 반드시 연습해야 합니다.

### SQL

**2~3문제** 출제됩니다.

- SELECT + JOIN + 서브쿼리 복합 문제
- DDL (CREATE, ALTER)
- DML (INSERT, UPDATE, DELETE)
- 집계함수 + GROUP BY + HAVING

### 소프트웨어 공학

**3~5문제**가 개념 용어를 묻습니다.

- 디자인 패턴 (싱글톤, 옵저버, 팩토리 등)
- 테스트 (화이트박스, 블랙박스, 단위, 통합)
- 개발 방법론 (애자일, 스크럼, XP)
- UML 다이어그램

### 네트워크/보안

**2~3문제**가 약어와 프로토콜을 묻습니다.

- OSI 7계층, TCP/IP
- 암호화 (대칭키, 비대칭키, 해시)
- 네트워크 공격 (DoS, SQL Injection, XSS)
- 프로토콜 (HTTP, FTP, SMTP, DNS)

### 데이터베이스

**2~3문제**입니다.

- 정규화 (1NF ~ BCNF)
- 트랜잭션 (ACID)
- 회복 기법, 병행 제어

## 영역별 학습 전략

### 코드 문제: 직접 손으로 트레이싱

코드 문제는 "이해"가 아니라 **"추적"**입니다. 빈 종이에 변수 값을 하나씩 써가며 따라가세요.

**자주 나오는 함정:**
- 후위 증감 연산자 (`i++` vs `++i`)
- 재귀함수의 호출 스택
- Java 오버라이딩에서 참조 타입 vs 실제 타입
- Python의 `//` (정수 나눗셈) vs `/` (실수 나눗셈)

### 용어 문제: 약어를 풀어서 외우기

"SOLID 원칙에서 L이 뭔지" 같은 문제가 나옵니다. 약어를 풀어서 외우면 기억에 잘 남습니다.

예시:
- **SOLID**: Single Responsibility, Open-Closed, Liskov Substitution, Interface Segregation, Dependency Inversion
- **ACID**: Atomicity, Consistency, Isolation, Durability

### SQL 문제: 실제로 쿼리 돌려보기

SQL 문제는 SELECT 결과를 적는 경우가 많습니다. 실제 DB에서 쿼리를 돌려보면서 연습하는 게 가장 효과적입니다.

## 채점 기준 알아두기

정처기 실기의 채점은 생각보다 **엄격**합니다.

- 단답형: **정확히 일치**해야 함 (대소문자, 띄어쓰기 주의)
- 서술형: **핵심 키워드**가 포함되어야 함
- 코드 실행 결과: 출력 형식까지 정확해야 함 (줄바꿈, 공백 포함)

> 모의고사를 풀 때도 "대충 맞으면 됐지"가 아니라, 정확한 답을 적는 연습을 하세요.
