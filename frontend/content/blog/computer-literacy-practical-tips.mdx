## 필기 붙고 실기에서 떨어지는 사람이 많습니다

컴활 1급 필기 합격률은 50–60%인데, 실기는 **30%대**예요. 필기는 이론만 외우면 되지만 실기는 엑셀과 액세스를 **직접 조작**해야 하기 때문이에요. 마우스 클릭 순서를 틀리거나, 함수 괄호를 하나 빠뜨려도 점수가 안 나옵니다.

필기와 실기를 같은 시험이라고 생각하면 안 돼요. 실기는 별개의 시험이라고 접근하는 게 맞습니다.

| 항목 | 필기 | 실기 |
|------|------|------|
| 형태 | 4지선다 60문항 | 컴퓨터 조작 (엑셀 + 액세스) |
| 시간 | 60분 | 90분 |
| 합격률 | 50–60% | 30%대 |
| 준비 기간 | 1–2주 | 3–4주 |

---

## 실기 시험 구성

실기는 크게 두 파트로 나뉘어요:

| 영역 | 배점 | 시간 배분 |
|------|------|----------|
| **엑셀** (스프레드시트 실무) | 60점 | 약 50–55분 |
| **액세스** (데이터베이스 실무) | 40점 | 약 35–40분 |

엑셀에서 많이 따야 합격이 유리해요. 액세스는 익숙하지 않은 사람이 많아서 여기서 점수를 많이 잃습니다.

---

## 엑셀 영역 — 배점이 가장 큰 파트

### 출제 유형

| 유형 | 배점 비중 | 난이도 |
|------|----------|--------|
| 함수 작성 | 높음 | 중–상 |
| 조건부 서식 | 중간 | 중 |
| 필터/정렬 | 낮음 | 하 |
| 차트 생성/편집 | 중간 | 중 |
| 매크로/VBA | 높음 | 상 |
| 피벗 테이블 | 중간 | 중 |

### 핵심 함수 — 이것만은 확실히

필기에서 외운 함수를 실기에서는 **직접 입력**해야 해요. 자주 나오는 함수:

- **VLOOKUP**: 거의 매회 나옴. 네 번째 인수(정확/근사) 구분 중요
- **INDEX + MATCH**: VLOOKUP보다 유연. 좌측 검색이 필요할 때
- **SUMIFS / COUNTIFS**: 여러 조건 합계/개수. S가 붙으면 복수 조건
- **IF + AND/OR**: 중첩 IF를 쓰는 경우가 많음
- **DSUM / DCOUNT / DAVERAGE**: D함수는 데이터베이스 함수. 조건 범위를 따로 지정

<Highlight color="red">실수 주의: 함수명 뒤에 괄호를 빠뜨리거나, 셀 범위를 잘못 지정하면 0점이에요. 입력 후 Enter 전에 한 번 더 확인하세요.</Highlight>

### 매크로/VBA — 포기하면 아쉬운 영역

2024년 개편 이후 VBA 배점이 올라갔어요. 기본적인 수준만 나오지만 아예 안 보고 가면 손해가 커요.

자주 나오는 VBA 패턴:
- `Range("A1").Value` — 셀 값 읽기/쓰기
- `For i = 1 To 10` — 반복문
- `If ~ Then ~ Else` — 조건문
- `Cells(행, 열)` — 셀 참조

코딩을 처음 보는 분도 이 4가지 패턴만 기출에서 연습하면 기본 점수는 챙길 수 있어요.

## 액세스 영역 — 낯설지만 패턴이 있습니다

액세스(Access)를 한 번도 안 써본 분이 많을 거예요. 그래서 실기 합격률이 낮은 거기도 해요.

### 출제 유형

| 유형 | 배점 비중 |
|------|----------|
| 테이블 생성/수정 | 중간 |
| 쿼리 작성 (SQL 포함) | 높음 |
| 폼 생성/편집 | 중간 |
| 보고서 생성 | 낮음–중간 |

### 핵심 포인트

**쿼리가 가장 중요해요.** SQL 기반 쿼리를 직접 작성하거나, 디자인 보기에서 조건을 설정하는 문제가 나옵니다.

자주 나오는 SQL 패턴:
- `SELECT ~ FROM ~ WHERE` 기본 조회
- `GROUP BY ~ HAVING` 그룹 조건
- `INNER JOIN ~ ON` 테이블 결합
- `Between ~ And` 범위 조건
- 집계 함수: `Sum()`, `Count()`, `Avg()`

<Highlight color="amber">팁: SQLD나 정처기 실기를 준비한 적 있다면 액세스 SQL은 수월하게 느껴질 거예요. 문법이 거의 같거든요.</Highlight>

### 액세스 준비법

교재만 읽으면 안 돼요. **액세스를 설치하고 직접 만져봐야** 합니다.

1. Microsoft 365 체험판으로 액세스 설치
2. 기출 문제의 액세스 파일을 다운받아서 직접 풀기
3. 테이블 → 쿼리 → 폼 → 보고서 순서로 연습
4. 특히 쿼리 디자인 보기와 SQL 보기를 왔다 갔다 하면서 감을 익히기

## 시간 관리가 합격을 결정합니다

실기 90분은 생각보다 빠듯해요. 시간 배분을 정해두고 들어가세요.

| 시간 | 할 일 |
|------|------|
| 0–5분 | 문제 전체 훑기. 쉬운 문제 먼저 파악 |
| 5–55분 | 엑셀 영역 풀기 (50분) |
| 55–85분 | 액세스 영역 풀기 (30분) |
| 85–90분 | 저장 확인. 빠뜨린 문제 체크 |

<Highlight color="red">실기 시험에서 가장 치명적인 실수는 "저장을 안 하는 것"입니다. 풀다가 시간에 쫓겨 저장을 누르지 못하면 0점이에요. 풀 때마다 Ctrl+S 습관을 들이세요.</Highlight>

## 준비 기간 가이드

| 상황 | 엑셀 | 액세스 | 총 기간 |
|------|------|--------|--------|
| 엑셀 경험 있음 | 1–2주 | 2–3주 | 3–4주 |
| 엑셀 초보 | 2–3주 | 2–3주 | 4–5주 |
| 완전 비전공자 | 3주 이상 | 3주 이상 | 5주 이상 |

액세스에 시간이 더 필요한 이유는 단순해요 — 대부분의 사람이 액세스를 처음 써보기 때문이에요. 엑셀은 일상에서 써본 경험이 있지만 액세스는 아예 없는 경우가 많습니다.

---

## 정리

컴활 실기는 **직접 조작하는 시험**이라 교재만 읽어서는 합격이 어려워요. 엑셀 함수를 손으로 입력하고, 액세스를 실제로 열어서 쿼리를 만들어보는 연습이 핵심입니다.

엑셀에서 안정적으로 40점 이상을 확보하고, 액세스에서 20점 이상을 가져오는 게 합격 전략이에요.
