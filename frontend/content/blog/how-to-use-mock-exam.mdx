## 모의고사, 그냥 풀기만 하면 소용없다

자격증 시험을 준비할 때 많은 분들이 모의고사를 풀고는 "점수 확인"만 하고 넘어갑니다. 70점이면 "아 괜찮네", 50점이면 "아 망했다" — 이게 끝입니다.

하지만 **모의고사의 진짜 가치는 풀고 난 후에 있습니다.**

---

## 1단계: 시간을 재고 풀기

### 실제 시험처럼 풀어야 하는 이유

- **SQLD**: 50문항 60분 → 문항당 1분 12초
- **정처기 실기**: 20문항 150분 → 문항당 7분 30초
- **컴활 1급**: 60문항 60분 → 문항당 1분

시간 압박 속에서 푸는 것과 여유롭게 푸는 것은 완전히 다릅니다. 실제 시험에서 시간이 부족해서 떨어지는 경우가 생각보다 많습니다.

> 첫 모의고사는 시간 제한 없이, 두 번째부터는 반드시 시간을 재세요.

---

## 2단계: 오답 분석이 핵심이다

### 틀린 문제를 분류하세요

모의고사를 풀고 나면 틀린 문제가 나옵니다. 이걸 그냥 "아 맞네" 하고 넘기면 다음에도 똑같이 틀립니다.

틀린 문제는 세 가지로 분류됩니다:

1. **몰라서 틀린 것** — 개념 자체를 모르는 경우. 해당 주제를 교재에서 다시 학습.
2. **헷갈려서 틀린 것** — 비슷한 개념을 혼동한 경우. 비교표를 만들어서 정리.
3. **실수로 틀린 것** — 알지만 실수한 경우. 어떤 실수인지 기록.

### 오답노트 활용

오답노트에 모아둔 문제를 다시 풀어보세요. **2번 연속 맞추면 마스터**한 것으로 판단할 수 있습니다.

한 번 맞췄다고 빼버리면 안 됩니다. 시간이 지나면 다시 틀릴 수 있거든요.

[오답 노트 확인하기 →](/wrong-answers)

## 3단계: 카테고리별 집중 학습

모의고사를 3회 이상 풀면 패턴이 보입니다. "JOIN에서 계속 틀리네", "윈도우 함수가 약하네" — 이런 패턴이 나오면 해당 카테고리만 집중적으로 푸세요.

### 카테고리 무한 풀이 활용법

1. 약한 카테고리를 선택
2. 문제를 계속 풀기 (이미 푼 문제는 후순위로 밀림)
3. 정답률이 80% 이상 올라올 때까지 반복

전체 모의고사를 반복하는 것보다 **약한 영역을 집중 공격**하는 게 점수를 올리는 가장 빠른 방법입니다.

[카테고리별 문제 풀기 →](/solve)

## 4단계: 점수 추이 확인

모의고사를 여러 번 풀면 대시보드에서 **점수 추이**를 확인할 수 있습니다.

- 점수가 꾸준히 오르고 있다면 → 현재 학습법이 효과적
- 점수가 정체됐다면 → 약점 영역을 다시 분석
- 점수가 떨어졌다면 → 새로운 유형에 약한 것, 해당 유형 집중 학습

숫자로 확인하면 막연한 불안감이 줄어듭니다.

## 시험 일주일 전: 마무리 전략

### D-7 ~ D-3
- 새로운 개념 학습은 멈추기
- 기존에 틀렸던 문제만 반복
- 모의고사 1회/일 풀기

### D-2 ~ D-1
- 오답노트만 훑기
- 핵심 공식/약어 정리본 복습
- 일찍 자기

### D-day
- 시험 1시간 전: 아무것도 안 보기 (오히려 혼란스러워짐)
- 모르는 문제는 과감히 넘기기
- 마지막 10분은 검토 시간

## 모의고사 선택 팁

좋은 모의고사의 조건:

- **문제가 매번 바뀔 것** — 같은 문제만 반복하면 답을 외우게 됩니다
- **해설이 있을 것** — 왜 틀렸는지 이해해야 다음에 맞출 수 있습니다
- **실제 시험과 유사할 것** — 문항 수, 구성, 난이도가 실제와 비슷해야 합니다
- **오답 추적이 될 것** — 틀린 문제를 다시 풀 수 있어야 합니다
