## 컴활 필기, 왜 단기 합격이 가능한가

컴활 1급 필기는 합격률이 50~60%대예요. IT 자격증 중에서는 높은 편이고, 시험 범위도 정해져 있어서 **기출 패턴만 익히면** 1~2주 안에 합격할 수 있습니다.

다만 "쉽다"와 "준비 안 해도 된다"는 다른 말이에요. 준비 없이 들어가면 과목별 용어가 낯설어서 생각보다 당황하거든요. 짧은 시간이라도 전략적으로 쓰는 게 중요합니다.

| 항목 | 내용 |
|------|------|
| 시험 형태 | 4지선다 60문항 (과목당 20문항) |
| 시험 시간 | 60분 |
| 합격 기준 | 평균 60점 이상 + **과목별 40점 이상** |
| 과목 | 컴퓨터 일반 / 스프레드시트 일반 / 데이터베이스 일반 |

<Highlight color="red">주의: 평균 60점이 넘어도 한 과목이 40점 미만이면 과락입니다. 세 과목을 골고루 봐야 해요.</Highlight>

---

## 과목별 특징과 전략

### 1과목: 컴퓨터 일반 (20문항)

운영체제, 네트워크, 멀티미디어, 하드웨어 등 **IT 상식 느낌의 과목**이에요. 범위가 넓어 보이지만 기출에서 반복되는 용어가 한정되어 있습니다.

**자주 나오는 주제:**
- 운영체제 개념 (프로세스, 스레드, 가상메모리)
- 네트워크 기초 (OSI 7계층, TCP/IP, 프로토콜)
- 멀티미디어 파일 형식 (JPEG, PNG, MP3, AVI 차이)
- 컴퓨터 구조 (CPU, RAM, 캐시메모리)

**공부법:**
기출에서 나온 용어를 중심으로 외우세요. 교재를 처음부터 정독할 필요 없어요. 기출 3회분만 풀어보면 어떤 용어가 반복되는지 감이 옵니다.

이 과목은 20문항 중 12개만 맞추면 과락은 피하고, 14~15개 맞추면 다른 과목에 여유를 줄 수 있어요.

### 2과목: 스프레드시트 일반 (20문항)

엑셀 함수와 기능을 묻는 과목이에요. **실제로 엑셀을 열 필요는 없지만**, 함수가 어떤 결과를 내는지는 알아야 합니다.

**매회 거의 나오는 함수:**

| 함수 | 기능 | 예시 |
|------|------|------|
| VLOOKUP | 세로 방향 찾기 | `VLOOKUP(찾을값, 범위, 열번호, 0)` |
| IF | 조건 분기 | `IF(A1>60, "합격", "불합격")` |
| COUNTIF | 조건부 개수 | `COUNTIF(A1:A10, ">60")` |
| SUMIF | 조건부 합계 | `SUMIF(조건범위, 조건, 합계범위)` |
| INDEX/MATCH | 고급 찾기 | VLOOKUP의 유연한 버전 |
| LEFT/MID/RIGHT | 문자열 추출 | `LEFT("컴활", 1)` → "컴" |

**공부법:**
함수 10~15개만 확실히 외우면 이 과목은 거의 안정권이에요. 기출에서 함수 문제를 따로 모아서 반복하는 게 효율적입니다. 매크로/VBA 문제도 1~2개 나오는데, 기초적인 것만 나오니 기출 수준만 봐두면 됩니다.

### 3과목: 데이터베이스 일반 (20문항)

DB 이론을 묻는 과목이에요. SQL 쿼리도 나오지만 실행하는 게 아니라 **"이 쿼리의 결과는?"** 형태로 나옵니다.

**자주 나오는 주제:**
- 정규화 (1NF, 2NF, 3NF, BCNF)
- SQL 기본 (SELECT, WHERE, GROUP BY, JOIN)
- 트랜잭션 (ACID 특성)
- 키 종류 (기본키, 외래키, 후보키, 슈퍼키)
- ER 모델 (엔터티, 관계, 속성)

**공부법:**
SQLD를 준비한 적 있다면 이 과목은 거의 공짜예요. 아니더라도 SQL 기본 문법과 정규화 개념만 잡으면 절반 이상 맞출 수 있습니다.

## 일주일 학습 플랜

시간이 정말 없다는 전제로, **하루 2~3시간 기준** 일정이에요.

| 날짜 | 할 일 |
|------|------|
| **1일차** | 기출 1회분 풀어보기 (시간 재지 말고). 어떤 유형이 나오는지 파악 |
| **2일차** | 1과목(컴퓨터 일반) — 기출에서 틀린 용어 위주로 정리 |
| **3일차** | 2과목(스프레드시트) — 함수 10개 외우기 + 기출 함수 문제 풀기 |
| **4일차** | 3과목(데이터베이스) — 정규화, SQL, 키 개념 정리 |
| **5일차** | 기출 2~3회분 연속 풀기. 과목별 점수 체크 |
| **6일차** | 약한 과목 집중 보강. 과락 위험 과목이 있으면 거기에 시간 몰빵 |
| **7일차** | 오답만 훑기. 새로운 개념은 보지 말 것 |

> <Highlight color="green">핵심: 교재 정독보다 기출 반복이 훨씬 효율적이에요. 컴활 필기는 기출 재활용률이 높은 시험입니다.</Highlight>

## 자주 하는 실수

### "1과목은 상식이니까 안 봐도 되겠지"

IT 비전공자에게는 "상식"이 아닌 용어가 꽤 나와요. OSI 7계층, 프로세스 스케줄링, RAID 같은 건 한 번은 봐야 합니다. 과락 기준이 40점(8문제)이니까, 20문항 중 12개를 틀리면 과락이에요.

### "함수를 다 외워야 하나?"

아니에요. 기출에서 반복적으로 나오는 함수 10~15개만 확실히 알면 됩니다. VLOOKUP, IF, COUNTIF, SUMIF만 잘 알아도 스프레드시트 과목의 절반은 해결돼요.

### "SQL이 어려워서 3과목 포기하려고요"

3과목은 SQL "만" 나오는 게 아니에요. 정규화, 키 개념, ER 모델 같은 이론 문제도 많이 나옵니다. SQL을 완전히 모르더라도 이론 문제만 잘 맞추면 과락은 피할 수 있어요.

## 시험 당일 팁

- **60문항 60분**이니까 문제당 1분이에요. 모르는 문제에 3분 이상 쓰지 마세요
- 4지선다니까 확실히 아닌 보기부터 지워나가면 정답률이 올라가요
- CBT(컴퓨터) 시험이라 **남은 시간이 화면에 보입니다.** 20분 남았을 때 안 푼 문제 수를 체크하세요
- 찍더라도 빈칸으로 두지는 마세요. 4지선다는 찍어도 25% 확률이에요

## 필기 붙고 나서

필기에 합격하면 실기를 준비해야 하는데, 실기는 완전히 다른 시험이에요. 엑셀과 액세스를 **실제로 조작**해야 하고, 합격률도 30%대로 뚝 떨어집니다.

필기 합격 후 실기까지의 기간이 넉넉하지 않으니, 필기 합격 직후 바로 실기 준비에 들어가는 게 좋습니다. 필기에서 외운 함수 지식이 실기에서도 그대로 쓰이거든요.
