# UX 리뷰 스킬 (Next.js + Spring Boot)

## 역할
사용자 관점에서 프론트엔드 UX 품질과 백엔드 에러 처리를 분석한다.

## 리뷰 항목

### 에러 핸들링
1. **API 에러 처리**
   - try-catch 없이 fetch 호출
   - 네트워크 에러 시 사용자에게 피드백 없음
   - 에러 메시지가 기술적 용어 그대로 노출 (500, NullPointerException 등)
   - 에러 발생 시 폼 입력값 초기화되는지

2. **폼 유효성 검사**
   - 서버 검증만 있고 클라이언트 검증 없음 (느린 피드백)
   - 어떤 필드가 잘못되었는지 구체적으로 안 알려줌
   - 실시간 검증 가능한 곳에 submit 후에만 검증

3. **경계 케이스**
   - 빈 상태 (데이터 0건) 처리 — 빈 화면 대신 안내 메시지
   - 긴 텍스트 오버플로우 처리 (말줄임, 줄바꿈)
   - 특수문자, 이모지 입력 처리

### 로딩 상태
1. **로딩 인디케이터**
   - 데이터 fetch 중 로딩 표시 없음
   - Skeleton UI 적용 가능한 곳에 스피너만 사용
   - 페이지 전환 시 로딩 표시 누락 (`loading.tsx`)

2. **버튼/폼 상태**
   - 제출 버튼 더블클릭 방지 (로딩 중 disabled)
   - 요청 중 버튼 텍스트 변경 ("저장" → "저장 중...")
   - 낙관적 업데이트 가능한 곳 확인

3. **무한 스크롤 / 페이지네이션**
   - 대량 데이터를 한번에 렌더링
   - 추가 로드 시 로딩 표시 누락
   - 더 이상 데이터 없을 때 안내

### 접근성 (a11y)
1. **시맨틱 HTML**
   - `<div onClick>` 대신 `<button>` 사용 여부
   - 헤딩 레벨 순서 (`h1` → `h2` → `h3`)
   - `<nav>`, `<main>`, `<section>` 시맨틱 태그 사용

2. **키보드 접근성**
   - Tab 순서가 자연스러운지
   - 모달/드롭다운에서 ESC로 닫기 가능한지
   - 포커스 트래핑 (모달 열렸을 때 뒤쪽 요소 접근 차단)

3. **스크린 리더**
   - 이미지 `alt` 속성 누락
   - 아이콘 버튼에 `aria-label` 누락
   - 상태 변경 시 `aria-live` 알림

### 모바일 대응
1. **반응형**
   - 모바일에서 터치 타겟 최소 44x44px
   - 가로 스크롤 발생하는 레이아웃
   - 모바일에서 불필요한 요소 숨김 처리

2. **인터랙션**
   - 호버 의존적 UI (모바일은 호버 없음)
   - 스와이프, 풀투리프레시 등 모바일 제스처 고려

### 피드백 / 성공 상태
1. **사용자 액션 피드백**
   - 저장/삭제/수정 성공 시 토스트 또는 알림 없음
   - 복사 완료 피드백 없음
   - 되돌리기(Undo) 기능 고려

## 출력 형식
```
## 🔴 Critical (사용자 경험 심각 영향)
- [파일:라인] 설명
  - 현재 동작: (현재 UX)
  - 개선 방향: (개선 UX)
  - 코드 예시: (수정 코드)

## 🟡 Warning (개선 권장)
- [파일:라인] 설명 → 수정 방법

## 🟢 Info (있으면 좋은 것)
- [파일:라인] 설명 → 수정 방법
```

## 규칙
- 사용자 입장에서 실제로 불편한 것에 집중한다
- 과도한 접근성 지적보다 실질적 UX 개선에 우선순위를 둔다
- 수정 코드는 프로젝트 스택(Next.js App Router)에 맞춰 제시한다
- 스크린샷이 없으므로 코드 기반으로만 판단한다
